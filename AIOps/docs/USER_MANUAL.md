# AIOps 用户操作手册

## 系统概述

AIOps大数据集群智能运维平台是一个基于AI的运维分析系统，帮助运维人员快速发现和诊断集群问题。

---

## 访问系统

1. 打开浏览器，访问 `http://localhost:8080/`
2. 系统首页展示集群概览信息

---

## 功能模块

### 1. 集群概览

首页顶部展示当前集群的核心指标：

| 指标 | 说明 |
|------|------|
| 集群健康度 | 综合健康评分(0-100) |
| 节点状态 | 活跃节点/总节点数 |
| HDFS使用率 | HDFS存储使用百分比 |

#### 切换集群

点击顶部的集群按钮可切换不同集群：
- `bigdata-prod` - 生产集群
- `bigdata-test` - 测试集群

---

### 2. 组件健康分析

#### 操作步骤

1. 选择分析类型为"组件分析"
2. 选择服务类型：
   - HDFS
   - YARN
   - Spark
   - Kafka
   - Hive
   - Flink

3. 选择组件类型：
   - NameNode / DataNode
   - ResourceManager / NodeManager
   - Master / Worker
   - Broker

4. 输入指标数据：
   - **CPU使用率**: 0-1之间的小数，如0.75表示75%
   - **内存使用率**: 0-1之间的小数
   - **GC时间**: 毫秒数

5. 点击"开始分析"按钮

#### 分析结果解读

分析结果包含以下部分：

**健康状态**
- 🟢 **健康(healthy)**: 组件运行正常
- 🟡 **警告(warning)**: 存在潜在问题
- 🔴 **严重(critical)**: 需要立即处理

**诊断信息**
- 问题摘要
- 具体问题列表
- 根因分析

**优化建议**
- 建议类型和优先级
- 具体操作步骤

---

### 3. Spark作业分析

#### 操作步骤

1. 切换到"Spark作业分析"标签页
2. 输入作业信息：
   - **作业名称**: 作业标识
   - **执行时长**: 毫秒数
   - **Executor数量**: 执行器数量
   - **数据倾斜率**: 最大任务时间/中位数时间
   - **Shuffle比例**: Shuffle数据占比

3. 点击"开始分析"

#### 常见问题诊断

| 问题类型 | 说明 | 建议操作 |
|----------|------|----------|
| data_skew | 数据倾斜 | 对倾斜Key加盐处理 |
| shuffle | Shuffle数据量大 | 调整分区数，开启AQE |
| gc | GC时间过长 | 增加Executor内存 |
| memory | 内存溢出 | 调整内存配置 |

---

### 4. 告警管理

#### 查看告警

右侧告警列表展示当前未处理的告警：
- 🔴 **严重告警**: 红色标记
- 🟡 **警告告警**: 黄色标记

#### 生成测试数据

点击"生成测试数据"按钮可生成模拟数据用于测试。

## 最佳实践

### 1. 定期分析

建议定期对关键组件进行健康分析：
- NameNode: 每小时分析一次
- ResourceManager: 每小时分析一次
- 重要Spark作业: 每次执行后分析

### 2. 阈值调整

根据实际业务情况调整告警阈值：

```yaml
# thresholds.yml
component:
  memory-usage:
    warning: 0.80  # 内存警告阈值
    critical: 0.90  # 内存严重阈值
```

### 3. 问题跟踪

对于分析发现的问题：
1. 记录问题详情
2. 按优先级处理
3. 验证优化效果

## 常见问题

### Q: 分析结果返回"unknown"状态？

**A**: 可能原因：
1. 输入数据不完整
2. LLM服务未正确配置
3. 网络连接问题

解决方法：
1. 检查输入数据是否完整
2. 确认LLM配置正确
3. 查看服务日志

### Q: 如何使用真实的大模型？

**A**: 修改配置文件：

```yaml
aiops:
  llm:
    provider: openai  # 或 claude, local
    
    openai:
      enabled: true
      api-key: your-api-key
```

### Q: 如何添加新的集群？

**A**: 
1. 在MySQL中插入集群数据
2. 前端会自动识别新集群

### Q: 告警数据如何清除？

**A**: 
- Redis中的告警数据会自动过期
- 可通过API手动处理告警
